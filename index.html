Buddy — Privacy Policy

Yasakarï Productions  ·  Last updated: March 2026

Buddy is a congregation management tool developed by Yasakarï Productions. This policy explains precisely what data is collected, where it is stored, and how it is protected.

1. How Publisher Data Enters the App

Personal publisher information — including full names, date of birth, baptism date, gender, and congregation roles — is entered exclusively through an encrypted import file (BPUB format) or manually by an administrator. This data is never collected automatically and is not sourced from any online service.

2. What Stays on the Device Only

The following data in connection with reporting is stored exclusively in an AES-256 encrypted local database on the device and is never uploaded to any server:

  • First Name and Last Name
  • Date of birth
  • Baptism date
This data never leaves the device. It cannot be accessed remotely under any circumstances.

3. What Is Uploaded — and How It Is Protected

Two categories of data are synchronised to the cloud to allow multi-device use within the same congregation:

Contact information (names & phone numbers)
Names and phone numbers are uploaded to Firebase Firestore, but are encrypted with AES-256 before leaving the device. The data arrives in Firebase as a single opaque encrypted blob — no readable fields exist in the cloud. Additionally, the link between a publisher's identity and their contact record is never uploaded and exists only in the local database. This means that even with direct access to the Firebase data, it is not possible to associate a contact entry with a specific person without both the device's local database and the congregation's private encryption key.

Service activity reports
Monthly activity data (reporting activity and studies) is uploaded to Firebase and is also fully encrypted with AES-256 before upload. Only an internal anonymous identifier is stored alongside the encrypted record in the cloud — no name or personal detail is included in plaintext.

All cloud-stored data is encrypted using a congregation-specific AES-256 key held securely on-device. Data in Firebase is unreadable without this key.

4. Access Control

Only authenticated users belonging to the same congregation can access synced data. Access is role-based:

  • Role 4 (Secretary): Full access.
  • Role 3: (Service Overseer) Editable Territory page.
  • Role 2 / 2A: Contact information only.
  • Role 1:Presently: Disabled role

5. Data Sharing

Buddy does not sell, share, rent, or disclose any data to third parties. Data is never used for advertising, analytics, or any purpose beyond congregation management.

6. Third-Party Services

Buddy uses Firebase (Google) solely for authentication and encrypted cloud sync between devices within the same congregation. This is governed by Google's Privacy Policy.

7. Data Retention & Deletion

Data is retained as long as the congregation uses the app. A Role 4 administrator can delete any record at any time, which removes it from both the local database and Firebase.

8. Minors

Buddy is intended exclusively for adult congregation members. This application is not used by minors.

9. Changes to This Policy

If this policy changes materially, users will be notified via an in-app notice on next login.
